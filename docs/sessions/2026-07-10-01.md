- **Shape of the result (11 tasks, all S/M, vertically sliced):** P5.1 submit
  endpoint + per-submit lead capture + 24h per-brand reuse → P5.2 pipeline
  `kind`-branch + fixed version-stamped 12-prompt set + `llm_cache` upsert →
  P5.3 presence map + competitors (read-time, $0, deterministic) → P5.4/P5.5

  public methodology page (rendered from a generated artifact, never a
  hand-copy) → P5.11 the one operator-gated live/cost/deploy/launch-gate task.
- **Key design decisions** (full record in the Phase-5 preamble + assumptions):
  reuse `analyses` via a nullable `kind` column and the existing GET envelope
  (no new runner, no worker change, no new infra); KYC reused as-is so the
  DRY_RUN demo keeps a meaningful ~0.5 score; competitors computed from raw
  answers via a deterministic proper-noun heuristic (no LLM call on the free
  endpoint); `CHECKER_ENABLED=0` default keeps prod dark until go-live;
  weighted score / sampling / sentiment and the full Turkish generation engine
  explicitly deferred (the binary score is the roadmap's "honest placeholder").
- **Verification story.** Round-1 verify issues were repaired by an agent;
  round 2 found 1 blocker + 2 majors + 5 minors; the round-2 repair and
  round-3 re-verify hit the API session limit, so the orchestrator adjudicated
  all 8 findings against the docs (all accepted) and applied the fixes by
  hand. The blocker mattered: the 24h cache shares one `analyses` row across
  visitors, so the original single `analyses.email` column would let later
  visitors' emails **overwrite earlier leads** on hot cached brands and cache
  hits would vanish from demand counts — fixed by making leads/demand
  per-submit via a small append-only `checker_submissions` table (the one new
  table Phase 5 now adds). Majors: P5.11's demand-test metrics moved to that
  table; P5.10's generator/Makefile wiring correctly assigned to the
  infra-owned lane (+ design.md §2 gen-types reconciliation when built).
  Minors: gate citation corrected to 02-mvp.md §3, a parallelism-map
  contradiction removed, the ADR-ordering claim softened, cross-lane review
  notes added for `deploy/.env.example` / `app/config.py`.
- **Workflow infrastructure note (for future orchestrators).** The Workflow
  `args` parameter reached the script as a *string* (not an object), so
  `args.scratch` was `undefined` and agents briefly wrote scratch files to an
  `undefined/` directory inside the repo — caught, moved to the session
  scratchpad, tree verified clean (`git status`). Pass workflow args as real
  JSON values and sanity-check paths interpolated into agent prompts.
