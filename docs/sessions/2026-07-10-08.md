## 1. Session summary

1. **Diagnosis (inline, before any code):** beyondtech.com.tr serves an
   890-byte Vite SPA shell whose only visible text is "Beyond Technologies"
   — discovery fed KYC 21 characters, the LLM hallucinated. ALL real content
   (BAZNA/Tayron/Liftron, kamikaze UAV specs, TR+EN mission text) sits in
   the 458KB JS bundle as string literals (verified by extraction). Nike
   works because it server-renders 8.4k chars. Separately, the prompt
   templates were mad-libs ("What is the best solutions in Turkey?") and put
   generic industry terms in every slot.
2. **Round 1 (workflow implement→verify, commit `c8a1932`):** discovery
   harvests title/meta/OpenGraph everywhere, prefers content-ful same-domain
   links (about/product/… + Turkish hakkinda/urun/hizmet, diacritic-folded),
   and adds an SPA fallback — <800 visible chars ⇒ mine ≤3 same-origin JS
   bundles (same SSRF-guarded client, 2MB cap) for prose string literals
   (Turkish-safe filtering). KYC prompt: facts only, verbatim product names,
   empty over guessed (mock coupling preserved). Prompts: natural templates,
   products-first topics, new "makers" category. **Verifier caught a
   do-not-ship live failure the implementer's fixtures masked:** framework
   runtime literals (ASCII) filled the 20k cap before the company content
   (~48k into the bundle) — fixed by ranking non-ASCII (localized) literals
   first + tighter code filtering, then re-proved against the live site.
3. **Live rerun 1 (`58913428`, $0.014):** KYC now correct (Defense
   Technology / UAV+UGV / full BAZNA product line) — but it exposed
   round-2 issues: product MODEL names leaked into category slots ("Who are
   the leading BAZNA V10 manufacturers?"), comma-list industry read badly,
   empty locations → "worldwide" instead of Türkiye.
4. **Round 2 (workflow, commit `e120f56`, verdict "ship" zero defects):**
   question slots draw category topics only (keywords > services ≤4 words >
   industry first-segment; ≤5 words), products confined to ≤count/4
   brand-probe prompts naming product+company; empty KYC locations backfill
   deterministically from the ccTLD (com.tr → Türkiye, 10-entry map,
   never overrides reported locations).
5. **Live rerun 2 (`9e4b2746`, $0.017) — the operator-facing proof:**
   industry "Defense Technology, Autonomous Systems, Unmanned Systems";
   products BAZNA V7/V10/V15/F, LIFTRON, FEDAİ; locations Turkey (LLM found
   it in the recovered text this time); prompts include "Who are the leading
   unmanned aerial vehicles manufacturers in Turkey?" (the operator's exact
   shape) and two BAZNA brand probes; **score 0.0 → 0.1** (4/40 — honest:
   engines only acknowledge the company when probed by name).
6. Suite: **99 backend passed** (+9 tests vs session 9), ruff/mypy clean,
   both commits deployed (last-good `e120f56`), co-tenants unaffected.
   **Session live spend $0.045** (runs: $0.014 + $0.014 + $0.017);
   cumulative ≈ $0.10.
