## 1. Session summary

**Completed: the last key-free task. The frontend lint script no longer calls
`next lint`** (deprecated in Next 15.5, removed in Next 16) — old tech-debt
#10 is repaid and a future Next 16 bump is no longer lint-blocked.

- **The change (`fa13839`, 2 lines):** `frontend/package.json` lint script
  `next lint` → `eslint . --ext .js,.jsx,.ts,.tsx --max-warnings 0`, and
  `next-env.d.ts` added to `.eslintrc.json` `ignorePatterns` (Next
  regenerates that file and it must never be hand-edited, so it must never
  gate CI; the official codemod's generated config ignores it too).
- **Why the minimal diff** (research findings, sources in the workflow run):
  installed toolchain is next 15.5.20 / eslint 8.57.1 / eslint-config-next
  15.5.20. eslint-config-next 15.x has **no native flat-config export**
  (FlatCompat is the documented 15.x pattern) and the official
  `next-lint-to-eslint-cli` codemod's legacy-config conversion is currently
  buggy (vercel/next.js#85679). eslintrc still works under ESLint 9, and the
  only hard Next-16 blocker is the `next lint` call itself. So: keep ESLint
  8.57 + `.eslintrc.json`, change only the script — **zero dependency and
  zero lockfile churn** (`npm ci` everywhere is untouched), and absorb the
  flat-config + ESLint 9 move into the future Next 16 bump where it belongs
  (recorded as new tech-debt #16).
- **Proven zero-diff before the edit:** the investigate workflow ran the
  exact new command against the unchanged tree — exit 0, **0 errors /
  0 warnings across 29 files**, including 17 files `next lint` never linted
  (its default dirs are only app/pages/components/lib/src): all of `tests/`,
  `e2e/happy-path.spec.ts`, and the root `*.config.ts` files. Coverage
  strictly widened, nothing regressed.
- **Deliberate strictness:** `--max-warnings 0` makes warning-level findings
  CI-blocking (`next lint` failed only on errors). Chosen because Next 16
  also stops linting during `next build`, making this script the only lint
  gate — a future warning failure is a real gate, not a flake (documented in
  tech-debt #16 and the commit message).
- **Verification:** the verify workflow mirrored the CI frontend job's four
  steps exactly (`tsc --noEmit` 5.8s, `npm run lint` 3.3s with the
  deprecation banner gone, vitest run 20/20, `next build` 46s with Next's
  build-time lint still running clean off the kept `.eslintrc.json`) — all
  exit 0, tree clean except the intended edit. The adversarial reviewer
  returned **approve-with-nits** (no blocker/major; the `next-env.d.ts` nit
  was adopted, the rest recorded in tech-debt). **CI run 29062634057:
  5/5 green** — frontend (typecheck + lint + vitest + build), backend,
  contract, secrets, e2e.
- **Scope discipline:** Phase 5 untouched (gate still lacks P4.1 + P4.2); no
  dependency bumps smuggled in; ESLint 9 explicitly NOT adopted.

**Why this way:** the decision framework (ship fastest, minimize diff,
replaceable later) — the 2-line change removes the Next-16 blocker today;
the fuller modernization has a natural, already-recorded home.
